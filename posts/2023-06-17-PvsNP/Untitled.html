On Function and Measure Composition.

Here we review the most basic property of functions and function composition.

Let \(f: X \to Y\) and \(g: Y\to Z\) be functions. The composition \(g\circ f\) is defined to be the function $h:=(gf):X Z $ according to the rule \[x \mapsto g(f(x)).\]

Now here is the tricky question: “What resources are necessary to evaluate the function \(h: X\to Z\) ?”

Naively one might assume that only \(X, Z\) are required to evaluate \(h\). But what is \(h\)? We say \(h\) is a function according to the definition \(h:=g\circ f\). But the definition of \(h\) only provides an implicit function \(h: X\to Z\) since there is no reference to \(Y\). The explicit evaluation of \(h\) necessarily involves resources on \(Y\), i.e. \(h(x)=g(y)\) where \(y:=f(x)\).

Therefore we caution the reader against assuming that the resources of a function \(h: X\to Z\) are exclusively the domain and range \(X, Z\). Rather the definition of composition requires the intermediate \(Y\), and therefore \(h\) as defined above really depends on \(X,Z\) and \(Y\).

Here we are somehow tracking how the function is defined in measuring it’s complexity. If the composition \(h=g\circ f\) can be reduced using algebraic identities, then it’s sometimes possible to obtain representations of the composition \(h: X \to Z\) directly using only \(x,z\) variables. But this is exception and not the rule.

As we have discussed in previous posts, functions are special cases of correlation measures as arising in optimal transport. If \(\pi_{01}\) and \(\pi_{12}\) are correlations between measures on \(X,Y\) and \(Y,Z\), then there is a natural composition defined \[\pi':=\pi_{12}\circ \pi_{01}.\] But the previous observations apply, and we find \(\pi'\) is implicitly a correlation between measures on \(X,Z\), but explicitly the correlation depends on correlations between measures on \(X,Y,Z\). In otherwords, the intermediate \(Y\) cannot be forgotten when evaluating the resources required to evaluate and compute using the measures. In otherwords, we think the composition \(\pi'\) should have an entropy measure which is dependant on the composition.

That the entropy measure should be made dependant on the composition means distinguishing “\(\pi'\) the composition” from “\(\pi'\) the correlation on \(X, Z\)” (with no reference to \(Y\)). Here we need be cautious, because we are arguing that the entropy measures should be representation dependant. Some readers might reject this outright, but it seems natural if we are accounting for the resources required for actual computation. In practice, the function \(h=g \circ f\) is not an identity but a definition, and the definition of the composition \(g\circ f\) necessarily involves \(Y\).

Entropy and Complexity of Correlations

Question: How to measure the complexity of a correlation? Specifically, the resources required to generate/run the correlation?

Option 1. Define \(k(\pi'):=\log \# spt(\pi')\) when \(\pi'\) is a correlation measure between homogeneous measures on \(X,Y\). If \(\pi'\) is a bijection, then \(k(\pi')\) coincides with \(k(proj_X\# \pi')\), the complexity of the pushforward measure. If \(\pi'\) is not a bijection, then we see the support of \(\pi'\) necessarily increases. In the extreme case, we find \(k(\pi')\) equal to the log of the product \[\# spt(proj_X \# \pi') \times \# spt(proj_Y \# \pi').\] This is case where the correlation \(\pi'\) is like a product measure and the correlation is supported on the Cartesian product of the supports of \(proj_X \# \pi'\), \(\proj_Y \# pi'\).

Option 2. We could define \(k(\pi')\) as the entropy of \(\pi'\). Following [Baez, et.al] one is tempted to define \(k(\pi'):= k(proj_Y \# \pi') - k(proj_X \# \pi')\). This would mean \(k\) represents the relative entropy between the source and image measures. This definition of entropy is well-behaved (additive) with respect to measure composition.
